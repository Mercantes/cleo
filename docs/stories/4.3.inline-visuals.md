## Story

**As a** user,
**I want** to see charts and tables in chat when I ask about data,
**so that** I understand my finances visually without leaving the conversation.

## Acceptance Criteria

1. When Cleo's response contains quantitative data, render inline visual (chart or table)
2. Visual types: bar chart (comparisons), line chart (projections), table (category breakdown), pie chart (distribution)
3. AI returns structured metadata indicating visual type and data (JSON in response)
4. Component `ChatVisual` renders the correct visual based on metadata
5. Visuals are responsive: adjust to message width
6. Fallback: if visual fails, show data in formatted text
7. At least 3 question types generate visuals: "Quanto gastei por categoria?", "Como é minha projeção?", "Comparar mês atual com anterior"

## Dev Notes

### Visual Metadata Format

Embedded in assistant message content:
```
Aqui está o breakdown dos seus gastos por categoria:

<!--VISUAL:{"type":"pie","title":"Gastos por Categoria - Março 2026","data":[{"name":"Alimentação","value":1200,"color":"#EF4444"},{"name":"Transporte","value":800,"color":"#F59E0B"},{"name":"Moradia","value":2500,"color":"#8B5CF6"},{"name":"Lazer","value":450,"color":"#EC4899"},{"name":"Outros","value":650,"color":"#6B7280"}]}-->

A alimentação continua sendo sua maior despesa...
```

### Parsing Logic

```typescript
function parseVisuals(content: string): { text: string; visuals: VisualMetadata[] } {
  const visualRegex = /<!--VISUAL:(.*?)-->/g;
  const visuals: VisualMetadata[] = [];
  const text = content.replace(visualRegex, (_, json) => {
    try {
      visuals.push(JSON.parse(json));
    } catch { /* skip invalid */ }
    return '';
  });
  return { text: text.trim(), visuals };
}
```
