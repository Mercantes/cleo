## Story

**As a** authenticated user,
**I want** to connect my bank account via Open Finance,
**so that** my transactions are automatically imported into Cleo.

## Acceptance Criteria

1. "Conectar banco" button on dashboard triggers connection flow
2. Pluggy Connect Widget opens in modal/iframe
3. User selects bank, authenticates with their bank credentials
4. On success: bank connection saved to `bank_connections` table
5. Accounts fetched and saved to `accounts` table
6. Connection status displayed (active, error, outdated)
7. User can see list of connected banks in settings
8. Error handling: connection failures show user-friendly messages
9. Mobile-responsive widget integration

## Dev Notes

### Pluggy Connect Widget

The Pluggy Connect Widget handles bank selection and authentication. It's embedded via the `react-pluggy-connect` package.

```typescript
// Usage pattern
import { PluggyConnect } from 'react-pluggy-connect';

<PluggyConnect
  connectToken={accessToken}
  onSuccess={(data) => handleImport(data.item.id)}
  onError={(error) => handleError(error)}
/>
```

### Connection Flow Sequence

```
User clicks "Conectar banco"
→ POST /api/pluggy/connect-token → { accessToken }
→ Open PluggyConnect widget
→ User selects bank, authenticates
→ Widget callback: onSuccess({ item })
→ POST /api/pluggy/import { itemId: item.id }
→ Server: INSERT bank_connections, INSERT accounts
→ Show success message
```

### Database Tables Used

- `bank_connections` — stores Pluggy itemId, connector_name, status
- `accounts` — stores account details (name, type, balance)
