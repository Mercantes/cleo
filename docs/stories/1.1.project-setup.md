## Story

**As a** developer,
**I want** a Next.js project configured with TypeScript, Tailwind CSS and shadcn/ui,
**so that** we have a standardized technical foundation for all development.

## Acceptance Criteria

1. Next.js 14+ (App Router) initialized with TypeScript strict mode
2. Tailwind CSS configured with base design tokens (colors, typography, spacing)
3. shadcn/ui installed and configured with at least Button, Input, Card and Dialog
4. ESLint + Prettier configured with consistent rules
5. Vitest configured with at least 1 passing example test
6. Directory structure defined: `src/app/`, `src/components/`, `src/lib/`, `src/types/`
7. `package.json` with scripts: dev, build, test, lint, typecheck
8. `.env.example` file with all required environment variables documented

## Dev Notes

### Project Structure Reference

The complete project structure is defined in `docs/architecture/system-architecture.md` Section 9. For this story, create only the top-level directories — subdirectories will be created in their respective stories.

```
src/
├── app/                # Next.js App Router (pages + API routes)
├── components/
│   ├── ui/             # shadcn/ui components
│   └── layout/         # App layout components (future stories)
├── lib/
│   ├── supabase/       # Supabase clients (Story 1.2)
│   └── utils/          # Shared utilities
├── types/              # TypeScript type definitions
├── hooks/              # React custom hooks
└── stores/             # Zustand stores
tests/
├── unit/               # Vitest unit tests
├── integration/        # Testing Library component tests
└── e2e/                # Playwright E2E tests
```

### Tech Stack (this story)

| Tech | Version | Purpose |
|------|---------|---------|
| Next.js | 14+ | App Router, RSC, API routes |
| TypeScript | 5.x | Strict mode |
| Tailwind CSS | 3.x | Utility-first CSS |
| shadcn/ui | latest | Component library (New York style) |
| Vitest | latest | Unit testing |
| ESLint + Prettier | latest | Linting + formatting |

### Coding Standards

- **Absolute Imports:** Always use `@/` prefix (e.g., `@/lib/utils`)
- **Server vs Client:** Default to Server Components. Add `'use client'` only when needed
- **Naming:** Components PascalCase, hooks camelCase with `use`, lib kebab-case, types PascalCase, constants UPPER_SNAKE

### Design Tokens (Cleo Brand)

For `tailwind.config.ts`, use these base colors:
- Primary: Blue tones (trust, finance)
- Success: Green (`#22C55E`) — income, positive
- Danger: Red (`#EF4444`) — expenses, negative
- Warning: Amber (`#F59E0B`) — alerts
- Neutral: Gray scale for backgrounds and text

### Testing

- **Framework:** Vitest with jsdom environment
- **Location:** `tests/unit/` for unit tests
- **Setup file:** `tests/setup.ts` with `@testing-library/jest-dom`
- **Coverage target:** Not enforced in this story
- **Pattern:** Write at least 1 passing test to validate the setup
