## Story

**As a** user,
**I want** a chat interface to talk to Cleo,
**so that** I can ask questions about my finances in natural language.

## Acceptance Criteria

1. Page `/chat` with messaging interface (input at bottom, messages above)
2. User messages aligned right, Cleo responses aligned left (with avatar)
3. Text input with send button and Enter key support
4. Welcome message from Cleo with suggested questions ("Quanto gastei esse mês?", "Minhas assinaturas estão pesando?")
5. "Cleo está digitando..." indicator during processing
6. Auto-scroll to latest message
7. Chat history persisted in Supabase (`chat_messages` table — already in schema)
8. Responsive: full screen on mobile, panel layout possible on desktop

## Dev Notes

### Chat Message Types

```typescript
interface ChatMessage {
  id: string;
  role: 'user' | 'assistant';
  content: string;
  metadata?: Record<string, unknown> | null;
  created_at: string;
}
```

### State Management

Use React useState for chat messages (local state). No need for Zustand for MVP — messages are loaded from API on mount and updated locally.

```typescript
const [messages, setMessages] = useState<ChatMessage[]>([]);
const [isLoading, setIsLoading] = useState(false);
const [input, setInput] = useState('');
```

### Auto-scroll Pattern

```typescript
const messagesEndRef = useRef<HTMLDivElement>(null);
useEffect(() => {
  messagesEndRef.current?.scrollIntoView({ behavior: 'smooth' });
}, [messages]);
```

### Cleo Avatar

Use a simple emoji or SVG for Cleo avatar: `🤖` or a custom icon.

### Database Table

```sql
-- Already exists in schema:
CREATE TABLE chat_messages (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  role TEXT NOT NULL CHECK (role IN ('user', 'assistant')),
  content TEXT NOT NULL,
  metadata JSONB,
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```
