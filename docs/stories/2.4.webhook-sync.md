## Story

**As a** user with a connected bank,
**I want** my transactions to be automatically synced when new ones appear,
**so that** I always have up-to-date financial data without manual refresh.

## Acceptance Criteria

1. Webhook endpoint `POST /api/webhooks/pluggy` receives Pluggy events
2. HMAC signature validation on all incoming webhooks
3. Handle `item/updated` event: update bank_connection status
4. Handle `transactions/updated` event: trigger incremental sync
5. Incremental sync: only fetch transactions since `last_sync_at`
6. Update `bank_connections.last_sync_at` after successful sync
7. Auto-refresh: scheduled sync every 6 hours (via Pluggy item refresh)
8. Environment variable: `PLUGGY_WEBHOOK_SECRET` for HMAC validation
9. Webhook endpoint returns 200 OK quickly, processes async

## Dev Notes

### Webhook Event Format

```json
{
  "event": "item/updated",
  "data": {
    "itemId": "xxx-xxx-xxx",
    "status": "UPDATED"
  }
}
```

```json
{
  "event": "transactions/updated",
  "data": {
    "itemId": "xxx-xxx-xxx",
    "accountId": "yyy-yyy-yyy"
  }
}
```

### HMAC Signature Validation

```typescript
import { createHmac, timingSafeEqual } from 'crypto';

function verifySignature(body: string, signature: string, secret: string): boolean {
  const expected = createHmac('sha256', secret).update(body).digest('hex');
  return timingSafeEqual(Buffer.from(signature), Buffer.from(expected));
}
```

### Incremental Sync Strategy

1. Read `bank_connections.last_sync_at`
2. Fetch transactions from Pluggy with `from=last_sync_at`
3. Deduplicate + save (reuse sync.ts from Story 2.3)
4. Categorize new transactions
5. Update `last_sync_at = now()`

### Auto-Refresh (6h)

Pluggy supports item refresh scheduling. The webhook fires `item/updated` when refresh completes, triggering our incremental sync automatically.
