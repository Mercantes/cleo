## Story

**As a** authenticated user,
**I want** a responsive app layout with sidebar and navigation,
**so that** I can navigate between sections of the application.

## Acceptance Criteria

1. Authenticated layout with sidebar (desktop) + bottom navigation (mobile)
2. Navigation items: Dashboard, Chat, Transações, Projeções, Configurações
3. Header with user name, avatar and logout button
4. Responsive: sidebar collapses on mobile, bottom nav appears
5. Dashboard page with placeholder "Conecte seu banco para começar"
6. Skeleton loading states for page transitions
7. Reusable empty state component

## Dev Notes

### Component Architecture (from system architecture doc)

```
src/components/
├── layout/
│   ├── app-sidebar.tsx       # Desktop sidebar navigation
│   ├── bottom-nav.tsx        # Mobile bottom navigation
│   └── header.tsx            # Top header with user menu
├── ui/
│   ├── empty-state.tsx       # Reusable empty state
│   └── page-skeleton.tsx     # Page-level loading skeleton
```

### Layout Pattern (from architecture doc)

```typescript
// src/app/(app)/layout.tsx
export default async function AppLayout({ children }) {
  const supabase = await createClient();
  const { data: { user } } = await supabase.auth.getUser();

  if (!user) redirect('/login');

  // Fetch profile for display
  const { data: profile } = await supabase
    .from('profiles')
    .select('*')
    .eq('id', user.id)
    .single();

  return (
    <div className="flex h-screen">
      <AppSidebar user={profile} />
      <div className="flex-1 flex flex-col">
        <Header user={profile} />
        <main className="flex-1 overflow-auto p-4 md:p-6">{children}</main>
      </div>
      <BottomNav />
    </div>
  );
}
```

### Responsive Breakpoints

- **Mobile** (< 768px): Bottom nav visible, sidebar hidden, hamburger menu in header
- **Desktop** (>= 768px): Sidebar visible, bottom nav hidden

### Dependencies on Story 1.3

- Auth middleware protecting `(app)/*` routes
- `(app)/layout.tsx` with auth check
- signOut server action for logout button

### Coding Standards

- **Server Components:** Layout is a Server Component (fetches profile)
- **Client Components:** Sidebar, BottomNav, Header need `'use client'` for interactivity (pathname, dropdowns)
- **Absolute Imports:** Always use `@/` prefix

### Testing

- **Framework:** Vitest + Testing Library
- **Location:** `tests/unit/components/`
- **Focus:** Component rendering tests and navigation data validation
