## Story

**As a** user,
**I want** to upgrade to the Pro plan via a secure payment flow,
**so that** I can unlock unlimited features with a seamless checkout experience.

## Acceptance Criteria

1. Stripe Checkout session creation via `POST /api/stripe/checkout` with Pro plan price ID
2. Webhook endpoint `POST /api/stripe/webhook` handles: `checkout.session.completed`, `customer.subscription.updated`, `customer.subscription.deleted`, `invoice.payment_failed`
3. On successful checkout: update user tier to 'pro' in profiles table, store Stripe customer ID and subscription ID
4. On subscription cancellation: downgrade user tier to 'free', preserve data but enforce limits
5. Customer portal link via `GET /api/stripe/portal` for managing subscription (cancel, update payment method)
6. Stripe webhook signature verification using `STRIPE_WEBHOOK_SECRET`
7. Idempotency: duplicate webhook events do not cause duplicate tier changes
8. Error handling: failed payments trigger email notification (future) and grace period flag

## Dev Notes

### Stripe Checkout Flow

```typescript
// POST /api/stripe/checkout
const session = await stripe.checkout.sessions.create({
  customer: stripeCustomerId,
  mode: 'subscription',
  line_items: [{ price: process.env.STRIPE_PRO_PRICE_ID, quantity: 1 }],
  success_url: `${baseUrl}/settings?session_id={CHECKOUT_SESSION_ID}`,
  cancel_url: `${baseUrl}/upgrade?canceled=true`,
  metadata: { userId },
});
return NextResponse.json({ url: session.url });
```

### Webhook Signature Verification

```typescript
// POST /api/stripe/webhook
export async function POST(req: NextRequest) {
  const body = await req.text();
  const signature = req.headers.get('stripe-signature')!;

  let event: Stripe.Event;
  try {
    event = stripe.webhooks.constructEvent(body, signature, process.env.STRIPE_WEBHOOK_SECRET!);
  } catch {
    return NextResponse.json({ error: 'Invalid signature' }, { status: 400 });
  }
  // Handle event...
}
```
