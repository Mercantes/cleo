## Story

**As a** user,
**I want** Cleo to automatically detect my subscriptions and installments,
**so that** I know exactly how much I'm committed to pay each month.

## Acceptance Criteria

1. Service `src/lib/finance/recurring-detector.ts` analyzes transactions and identifies recurring patterns
2. Subscriptions detected: same merchant, same amount (+/- 5%), monthly frequency, >= 2 consecutive months
3. Installments detected: "X/Y" pattern in description or identical amounts in sequence with same merchant
4. Results saved in `recurring_transactions` table (schema already exists in DB)
5. API route `GET /api/recurring` returns active subscriptions and installments
6. Page `/subscriptions` listing active subscriptions with: name, amount, frequency, annual total
7. Page shows active installments with: name, installment amount, remaining, total committed
8. Total monthly commitments sum displayed at the top

## Dev Notes

### Detection Algorithm

```typescript
interface RecurringResult {
  merchant: string;
  amount: number;
  frequency: 'monthly' | 'weekly' | 'yearly';
  type: 'subscription' | 'installment';
  installments_remaining?: number;
  next_expected_date: string;
  occurrences: number;
}

// Subscription detection:
// 1. Group transactions by normalized merchant name
// 2. For each group with >= 2 entries:
//    a. Sort by date
//    b. Check if intervals are ~30 days (+/- 5 days)
//    c. Check if amounts are within 5% variance
//    d. If both pass → subscription

// Installment detection:
// 1. Match description with regex: /(\d+)\s*(?:\/|de)\s*(\d+)/i
// 2. OR: same merchant, exact same amount, sequential months
```

### Database Table

```sql
-- Already defined in architecture schema:
CREATE TABLE recurring_transactions (
  id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
  user_id UUID NOT NULL REFERENCES profiles(id) ON DELETE CASCADE,
  transaction_pattern TEXT NOT NULL,
  merchant TEXT NOT NULL,
  amount DECIMAL(14,2) NOT NULL,
  frequency TEXT NOT NULL CHECK (frequency IN ('monthly', 'weekly', 'yearly')),
  type TEXT NOT NULL CHECK (type IN ('subscription', 'installment')),
  installments_remaining INT,
  next_expected_date DATE,
  status TEXT NOT NULL DEFAULT 'active',
  created_at TIMESTAMPTZ NOT NULL DEFAULT now()
);
```
