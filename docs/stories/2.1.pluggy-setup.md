## Story

**As a** developer,
**I want** a Pluggy API client configured with authentication and token management,
**so that** the app can communicate with Pluggy to connect banks and fetch transactions.

## Acceptance Criteria

1. Pluggy API client (`src/lib/pluggy/client.ts`) with typed methods
2. Authentication flow: `POST /auth` with client_id + client_secret → access_token
3. Access token auto-renewal (expires in 2h, renew before expiration)
4. API route `POST /api/pluggy/connect-token` to generate Pluggy Connect Widget token
5. Environment variables: `PLUGGY_CLIENT_ID`, `PLUGGY_CLIENT_SECRET` configured in `.env.local`
6. Rate limiting awareness (100 requests/min)
7. Error handling with typed errors and retry logic for transient failures
8. Unit tests for client methods and token management

## Dev Notes

### Pluggy API Reference

- **Docs:** https://docs.pluggy.ai
- **Base URL:** `https://api.pluggy.ai`
- **Auth:** POST /auth with `{ clientId, clientSecret }` → `{ apiKey }` (valid 2h)
- **Rate Limit:** 100 req/min per client
- **Pagination:** `pageSize` max 500, response includes `totalPages`

### Key Endpoints

| Method | Path | Description |
|--------|------|-------------|
| POST | /auth | Get access token |
| POST | /connect_token | Generate Connect Widget token |
| GET | /items/{id} | Connection status |
| GET | /accounts?itemId={id} | List accounts |
| GET | /transactions?accountId={id}&from=&to=&pageSize=500 | List transactions |
