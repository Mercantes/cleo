## Story

**As a** user with a connected bank,
**I want** my transactions to be automatically imported and categorized,
**so that** I can see my financial history organized in Cleo.

## Acceptance Criteria

1. Transaction import triggered after bank connection (Story 2.2)
2. Paginated fetch: handle 500+ transactions per account
3. Deduplication by `pluggy_transaction_id` — no duplicate entries
4. Transactions saved to `transactions` table with all fields mapped
5. AI categorization: batch categorize transactions via Claude API
6. Categories mapped from `categories` table (12 seed categories)
7. Import progress indication (loading state with count)
8. Error handling: partial import saves successful transactions, reports failures
9. Transaction sync logic extracted to `src/lib/pluggy/sync.ts` for reuse

## Dev Notes

### Transaction Field Mapping

| Pluggy Field | Cleo Field | Notes |
|-------------|------------|-------|
| id | pluggy_transaction_id | Unique dedup key |
| description | description | Raw bank description |
| amount | amount | Absolute value, sign from type |
| date | date | ISO date string |
| type | type | DEBIT→debit, CREDIT→credit |
| category | raw_category | Pluggy's own category |
| paymentData.receiver?.name | merchant | When available |

### AI Categorization Prompt Pattern

```
Categorize these bank transactions into one of these categories:
- Alimentação, Transporte, Moradia, Saúde, Educação, Lazer,
  Compras, Assinaturas, Receita, Transferência, Investimentos, Outros

For each transaction, return the category and confidence (0-1).

Transactions:
1. "IFOOD *IFOOD" R$45.90 (debit) → Alimentação (0.95)
2. "UBER *TRIP" R$23.00 (debit) → Transporte (0.90)
...
```

### Deduplication Strategy

Use `pluggy_transaction_id` as the unique key. On conflict:
- Do NOT update existing transactions (user may have edited category)
- Skip silently and count as `skipped`
