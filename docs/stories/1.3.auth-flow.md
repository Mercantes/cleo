## Story

**As a** user,
**I want** to sign up and log in with email/password or Google,
**so that** I can securely access my financial data.

## Acceptance Criteria

1. Signup page with form: name, email, password (validation with Zod)
2. Login page with email/password
3. "Continuar com Google" button functional via Supabase Auth (OAuth)
4. Redirect to `/dashboard` after successful login
5. Redirect to `/login` when accessing protected routes unauthenticated
6. Next.js middleware protecting authenticated routes (`(app)/*`)
7. Logout functionality
8. Error handling: duplicate email, wrong password, OAuth failure
9. Loading states on action buttons

### Auth Flow (from architecture doc)

**Email/Password Signup:**
1. User fills form → `supabase.auth.signUp({ email, password, options: { data: { full_name } } })`
2. Supabase creates user in `auth.users`
3. DB trigger `on_auth_user_created` auto-creates profile in `profiles` table
4. Redirect to `/dashboard`

**Google OAuth:**
1. User clicks "Continuar com Google"
2. `supabase.auth.signInWithOAuth({ provider: 'google', options: { redirectTo: '/auth/callback' } })`
3. Supabase redirects to Google → user authorizes → redirects back to `/auth/callback`
4. Callback route exchanges code for session
5. DB trigger creates profile
6. Redirect to `/dashboard`

### Middleware Pattern

```typescript
// Protected route patterns
const protectedPaths = ['/dashboard', '/chat', '/transactions', '/projections', '/settings'];
const authPaths = ['/login', '/signup'];
```

### Google OAuth Prerequisites

Google OAuth requires configuration in:
1. Supabase Dashboard → Authentication → Providers → Google
2. Google Cloud Console → OAuth 2.0 credentials

**For development:** Email/password auth works immediately. Google OAuth can be tested after configuring credentials.
