## Story

**As a** user with imported transactions,
**I want** to view, filter, and search my transactions,
**so that** I can understand my spending patterns and find specific transactions.

## Acceptance Criteria

1. Transaction list page at `/transactions` with paginated data
2. API route `GET /api/transactions` with filters (category, bank, date range, page)
3. Filter by: category, bank account, date range (from/to)
4. Search by description or merchant name
5. Each transaction shows: date, description, amount (colored: green income, red expense), category icon+name
6. Pagination: 50 items per page, load more button
7. Empty state when no transactions (with link to connect bank)
8. Mobile-responsive: card layout on mobile, table on desktop
9. Loading skeleton while fetching data

## Dev Notes

### API Query Pattern

```typescript
let query = supabase
  .from('transactions')
  .select('*, categories(*)')
  .eq('user_id', user.id)
  .order('date', { ascending: false })
  .range((page - 1) * pageSize, page * pageSize - 1);

if (category) query = query.eq('category_id', category);
if (bank) query = query.eq('account_id', bank);
if (from) query = query.gte('date', from);
if (to) query = query.lte('date', to);
if (search) query = query.or(`description.ilike.%${search}%,merchant.ilike.%${search}%`);
```

### BRL Currency Formatting

```typescript
const formatCurrency = (amount: number): string =>
  new Intl.NumberFormat('pt-BR', { style: 'currency', currency: 'BRL' }).format(amount);
```
